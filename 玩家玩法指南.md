# 《生灵契约》玩家玩法指南

本模组是一套以随机阵营、感染生物对抗、共享资源和缩圈终局为核心的多人玩法。每局开始后，玩家会被随机分队，并由队伍共同发展感染阵营、强化异变能力，最终争夺末地终局的胜利。

## 快速开始

1. 管理员在模组配置页面设置队伍数量和感染模组池。
2. 管理员执行 `/contract game start` 开始游戏。
3. 按 `Tab` 确认队伍颜色和感染模组，按 `B` 打开队伍背包。
4. 击杀生物获取“升华”；玩家背包与队伍背包中的升华总数就是可用异变点数（MP）。
5. 按 `U` 打开阵营异变树，消耗对应数量的升华购买阵营强化。
6. 在不断缩小的边界内发展、战斗，并寻找位于边界中心附近、地下约 `Y=-90` 的末地要塞。
7. 边界边长小于 `50` 格后，末地传送门自动开启。击杀诡异末影龙，或成为最后存活的队伍即可获胜。

## 游戏目标与胜利条件

满足以下任意条件，玩家所在队伍立即获胜：

- 本局参赛玩家击杀末地中的诡异末影龙。
- 场上只剩一支仍有非旁观玩家的队伍。

胜利后，全服会收到文字、标题和音效播报，本局游戏随即结束。

## 开局、分队与感染阵营

执行开始指令时，所有处于生存或冒险模式的在线玩家会被随机分队。每支队伍会从感染模组池中获得一个不同的感染模组，实际队伍数量不会超过以下三者中的最小值：

- 当前参赛玩家数量。
- 配置的随机队伍数量。
- 感染模组池中已加载的模组数量。

默认感染模组池如下：

| 感染模组 | 模组 ID |
| --- | --- |
| Caerula Arbor | `caerula_arbor` |
| 真菌感染：孢子 | `spore` |
| Phayriosis Reborn: CLASH OF THE TITANS `0.3.2.3` | `phayriosisreborn` |
| 幽谧部落 | `sculkhorde` |

管理员可从“模组列表 -> Life Contract -> 配置”进入配置页面，调整队伍数量，并直接勾选当前已加载的模组作为感染模组池。

### 队伍规则

- 同队玩家的名称颜色相同，不同队伍使用不同颜色。
- 按住 `Tab` 可以查看所有在线玩家的队伍颜色和其感染模组 ID。
- 同队玩家之间无法互相造成伤害。
- 与本队感染模组同属一个阵营的生物不会攻击本队玩家。
- 本队玩家、契约生物和感染生物共享同一份阵营异变进度。
- 本局开始后才加入服务器的玩家默认成为旁观者，可选择加入当前人数最少的队伍，也可继续旁观。

## 常用按键与界面

| 按键 | 功能 |
| --- | --- |
| `B` | 打开队伍背包 |
| `U` | 打开阵营异变树 |
| `Tab` | 查看按队伍着色的玩家名称和感染模组 |

默认 HUD 位于屏幕左侧，会显示队伍编号、当前感染模组和在线队友。队友高亮默认开启，附近队友会显示绿色轮廓。

按键可在 Minecraft 的“选项 -> 控制 -> 按键绑定 -> 生灵契约”中重新设置。

## 队伍背包

按 `B` 可随时打开一个全队共享的 `54` 格背包。

- 同队玩家看到并操作的是同一份物品。
- 背包内容会保存，并实时同步给正在使用背包的队友。
- `Shift+左键` 可在个人背包和队伍背包之间快速移动物品。
- 放入队伍背包的兼容矿石、粗矿和生肉会自动烧炼或烹饪，不消耗燃料。
- 自动处理包括原版铁、金、铜、远古残骸、常见生肉和部分兼容模组矿物。

队伍背包既是公共仓库，也是全队的自动熔炼区。重要装备仍建议分配清楚，避免多名玩家同时取用同一物品。

## 升华与异变点数

### 升华

- 每个生物死亡时都会掉落 `1` 个“升华”。
- “升华”名称呈动态金色，单个物品栏最多堆叠 `1024` 个。
- 升级异变词条时，优先消耗点击升级玩家背包中的“升华”，不足部分再从队伍背包扣除。

### 异变点数（MP）

- MP 不是独立保存的点数；`1` 个“升华”就等于 `1 MP`。
- 可用 MP 为当前玩家主物品栏与所在队伍公共背包中的“升华”数量之和。
- 队伍背包中的“升华”由全队共享，玩家个人背包中的部分只计入该玩家发起的升级。
- 表中费用是购买该等级时需要消耗的“升华”数量。例如“锋刃”从 Lv.II 升到 Lv.III 会消耗 `20` 个升华，即 `20 MP`。

### 解锁规则

- T1 无前置等级要求。
- T2 要求当前已购买的词条总等级至少为 `5`。
- T3 要求当前已购买的词条总等级至少为 `12`。
- 解锁 T2/T3 只检查当前总词条等级，不检查当前可用 MP。
- “群体协同：蜂群”和“基因重构：巨兽”互斥，一旦选择其中一个，就不能再购买另一个。

## 核心物品

### 生灵契约

手持“生灵契约”右键一只非己方、非玩家生物，可令其永久加入自己的阵营。契约生物会跟随并协助攻击其他阵营，队友无法误伤它。

使用代价：

- 玩家永久失去目标当前生命值 `3%` 的最大生命值。
- 若支付后玩家最大生命值会低于 `1` 点，则无法签订契约。
- 物品成功使用后消失。

工作台布局：

```text
钻石  铁锭  煤炭
铁锭  青金石 铁锭
红石  金锭  铁锭
```

### 跟随之杖

- 右键一只己方怪物，使其紧紧跟随自己。
- 同时只能绑定 `1` 只；绑定新目标会自动解除旧目标。
- `Shift+右键空气` 可主动解除当前跟随。
- 绑定期间，玩家的饥饿消耗额外增加原版的 `10%`，即总消耗约为原版的 `110%`。
- 跟随之杖不会被消耗。

工作台布局：

```text
空    铁锭  空
空    红石  空
空    铁锭  空
```

### 生物蛋

生物蛋没有常规合成配方，通常由服务器玩法或管理员提供。

- 空生物蛋 `Shift+右键` 非玩家生物，可以将其捕获。
- 装有生物的蛋右键方块可在方块旁释放生物。
- 右键空气可在视线指向的位置释放，最大瞄准距离为 `64` 格。
- 释放后的生物会保留类型、名称和生命值，并成为释放者的己方生物。
- 每个生物蛋只能存放一只生物，释放后物品消失。

### 肉泥与孢子炸弹

- `4` 份受支持的感染生物材料可无序合成为 `1` 个肉泥。
- 工作台铺满 `3x3` 的 `9` 个肉泥，可合成 `1` 个孢子炸弹。
- 右键投掷孢子炸弹，落点周围约 `5` 格内的玩家会获得 `3` 秒缓慢 III 和虚弱 I。
- 孢子炸弹不区分阵营，也可能影响投掷者和队友。
- 孢子炸弹最多堆叠 `16` 个。

### 组队器

组队器没有常规合成配方，通常用于管理员组织的自定义玩法。

- 右键玩家：令目标加入自己的队伍并共享感染阵营。
- `Shift+右键` 玩家：将目标移出当前队伍。
- 正式对局开始时仍会按照管理员配置重新随机分队。

## 死亡、复活与旁观

当一名队友处于待复活状态时，己方玩家击杀一名敌队玩家会打开复活选择界面。

- 可从仍在等待复活的在线队友中选择一人。
- 被选中的队友会在击杀者身边复活，恢复全部生命并清除当前状态效果。
- 也可以在界面中放弃本次复活机会。
- 玩家死亡后若原定复活点位于世界边界外，会自动移动到边界内侧的安全地表。
- 若所有队员都成为旁观者，该队会被视为已淘汰。

中途加入进行中对局的玩家会先进入旁观模式，并在聊天栏看到两个可点击选项：加入人数最少的队伍，或保持旁观。

## 边界、地下世界与末地终局

### 世界边界

- 游戏开始时，主世界边界边长为 `600` 格。
- 边界中心会锁定到距离开局位置最近的末地要塞传送门。
- 所有参赛玩家会被迁移到边界内部，末地传送门始终保持在缩圈范围内。
- 玩家即使位于或能够触及边界外，也可以正常挖掘和放置方块。
- 专用服务器的出生点保护默认关闭。

### 深层世界

- 主世界最低可建造高度为 `Y=-128`。
- 深层区域会继续生成适配后的矿物，适合在 `Y=-64` 以下采矿。
- 末地要塞及传送门房间位于约 `Y=-90`，请准备足够的照明、方块和挖掘工具。

### 末地传送门与 Boss

- 开局时，传送门框架会预先填入 `11` 颗末影之眼，但不会立即开启。
- 当边界边长小于 `50` 格时，最后一颗眼自动补齐，传送门随即开启。
- 玩家进入末地后，末地边界会固定包围主岛，中心为 `(0, 0)`、边长为 `300` 格，即在主岛四周预留约 `50` 格空间。
- 第一名玩家进入末地后，原版末影龙和原版末影龙血条会被移除，并生成 `phayriosis:converted_dragon` 作为诡异末影龙。
- 末地主岛中央会同时生成 `3` 只诡异末影人。
- 旧版 Phayriosis 阵营生物仅允许出现在末地；它与感染池中的 Phayriosis Reborn `0.3.2.3` 是不同阵营来源。
- 对诡异末影龙取得击杀归属的本局玩家，会为自己的队伍直接赢得比赛。

## 联动与额外规则

### Xaero 小地图

本模组不提供独立小地图。加载 Xaero 小地图后：

- 世界边界会以红色线条绘制在小地图上。
- 雷达不会显示非同队玩家的位置。
- 同队玩家仍可正常显示。

### FTB Ultimine

以下任一情况存在时，玩家无法使用连锁挖矿：

- 玩家正处于受击状态。
- 距离任意存活的非同队玩家不超过 `10` 格。

敌人离开范围且受击状态结束后，连锁挖矿会恢复正常。

### 其他战斗规则

- 玩家受到非玩家来源的普通伤害时，伤害会降低至原来的 `40%`；玩家对玩家伤害不受此减伤影响。
- 超过原版护甲上限的额外护甲仍能继续提供减伤，最高总减伤为 `95%`。
- 己方感染生物或契约生物无法伤害本队玩家，本队玩家也无法直接攻击己方生物。

## 金属锭转换经验

转换会直接消耗个人背包中的所有可转换物品，并给予经验“等级”，不是普通经验点。执行前请先将不想转换的材料放入队伍背包或其他容器。

| 物品 | 每个获得等级 |
| --- | ---: |
| 铁锭 | 1 |
| 铜锭 | 1 |
| 金锭 | 10 |
| 下界合金碎片 | 20 |
| 钻石 | 100 |
| 绿宝石 | 1000 |
| 下界合金锭 | 50000 |

## 普通玩家指令

以下指令无需管理员权限：

| 指令 | 作用 |
| --- | --- |
| `/contract hud` | 开关左侧生灵契约 HUD |
| `/contract_hud` | `/contract hud` 的快捷别名 |
| `/contract highlight` | 开关队友高亮 |
| `/contract ingot_to_exp` | 开关个人的金属锭自动转经验选项 |
| `/contract convert_ingots` | 立即转换个人背包中的所有可转换物品 |
| `/tp <队友名称>` | 无需权限传送到在线队友身边 |
| `/contract team tp <队友名称>` | 使用生灵契约指令传送到在线队友身边 |
| `/contract game join_low_team` | 中途加入正在进行的游戏，并进入当前人数最少的队伍 |
| `/contract game stay_spectator` | 保持旁观模式，不加入当前游戏 |

普通玩家使用 `/tp` 时只能输入一个在线队友名称。不能传送到自己、敌队玩家或任意坐标，也不能使用其他 `/tp` 语法；管理员仍可使用原版完整传送指令。

## 管理员指令

以下指令需要管理员权限等级 `2`：

| 指令 | 作用 |
| --- | --- |
| `/contract game start` | 自动随机分队、分配感染模组、锁定末地要塞并开始游戏 |
| `/contract game pause` | 暂停当前游戏状态 |
| `/contract game resume` | 继续已暂停的游戏 |
| `/contract game stop` | 停止当前游戏 |
| `/contract game status` | 查看游戏是否进行中、是否暂停以及已进行时间 |
| `/contract pool add <modid>` | 将指定模组 ID 加入感染模组池 |
| `/contract pool remove <modid>` | 从感染模组池移除指定模组 ID |
| `/contract pool list` | 查看当前感染模组池 |
| `/contract team split <队伍数>` | 随机分队，并尽量保留各队现有感染模组 |
| `/contract team random_split <队伍数>` | 随机分队，并从当前池中分配感染模组 |
| `/contract team set_mod <玩家> <modid>` | 为目标玩家及其所在队伍指定感染模组 |
| `/contract admin list` | 查看当前在线队伍、队员和感染模组 |
| `/contract spawn_sentinel <队伍编号>` | 在执行者位置生成带指定队伍编号名称的守卫实体 |

开始游戏前，建议管理员先执行 `/contract pool list`，确认池中至少有一个当前已加载的感染模组。若找不到末地要塞、没有可参赛玩家，或池中没有已加载模组，游戏不会开始。
